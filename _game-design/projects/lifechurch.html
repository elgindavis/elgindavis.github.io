Life.Church Software Engineering Internship

In my third year of university, I worked as a Software Engineering Intern for
7 months at Life.Church in Edmond, Oklahoma.

Developed web-based analytics applications with ​React JS​, ​Redux​, and ​Highcharts
● Set up foundation for Continuous Integration/Continuous Deployment pipelines for the
production website of Life.Church using the ​Google Cloud Engine​ and ​Kubernetes,​ and
providing ​Docker ​image implementations
● Implemented UX/UI changes and feature additions in ​HAML, CoffeeScript, Ruby on

● Configured CI/CD pipelines for the Life.Church mobile Android and iOS applications
● Constructed application templates to run Kubernetes on Google Cloud Platform
● Created microservices for a large-scale system in ​Ruby on Rails
● Designed a new web desktop interface in ​Figma​ for the Church Online host platform

Context

The settings area of trivago’s mobile apps had been long since outdated, according to my own personal assessment of the app and the input from the product managers and other mobile designers. The area was visually and conceptually disorganized, and its functionality did not match the users’ needs with respect to accessibility of important information.

In addition to this, I found that in my research and prior research done by the User Research team had done that the overall app felt very transactional and impersonal for the users, and when I was tasked with the complete redesign of the settings area as my main independent project for the first two months of the summer, I sought to improve the in-app experience by giving it a more solid character and personality, as well as solving the more technical issues at hand.
